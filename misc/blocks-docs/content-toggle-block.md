# Content Toggle Block Documentation

## Overview
The **Content Toggle Block** lets visitors switch between multiple pieces of content using pill-style toggles. Each toggle can show any Gutenberg blocks (headings, paragraphs, CTAs, media) and transitions are powered by GSAP for smooth fades/slides. Perfect for “Services vs. Approach” comparisons, feature matrices, or plan breakdowns.

## Adding the Block
1. Insert the “Content Toggle” block from the inserter.
2. Use the sidebar to manage toggle labels. Three demo pills are added automatically.
3. Select each toggle to edit its inner content via the nested block area below the pills.

## Toggle Management
- **Add / Remove** – Buttons in the sidebar let you append or delete toggles.
- **Reorder** – Move toggles up or down to set the navigation order.
- **Active Toggle** – Choose which pill loads as active on initial render.
- **Labels** – Update the pill label text directly in the sidebar.

## Styling Controls
- **Pill Colors** – Customize background, text, and border colors for both inactive and active states.
- **Typography** – Set font size and weight for the pill labels.
- **Spacing** – Control pill padding (per side) and the gap between pills.
- **Shape** – Adjust pill border radius for square, rounded, or pill-shaped buttons.
- **Position** – Display pills above or below the content container.

## Content Area Settings
- **Container Mode** – Full width or constrained with responsive max-width sliders.
- **Background & Padding** – Define the content panel background color, padding, and border radius.
- **Animation** – Configure GSAP duration and easing curve for toggle transitions.

## Best Practices
- Keep pill labels short (1–2 words) for best wrapping.
- Use contrasting active colors so visitors understand which panel is visible.
- Group related content in each toggle; avoid stacking too many sections per panel.
- Test on mobile to ensure pills remain tappable and pill gap isn’t too wide for small screens.

# Adaire Blocks v1.2.4 Release Notes

Version 1.2.4 adds custom block icons for the recently added blocks (Hero 1, Image Composition, Info Grid family, Card Scroll, Footer) and fixes block picker display and free-plugin categorization.

## Summary

This release focuses on visual consistency and editor experience: new SVG icons are integrated for nine blocks, block picker (inserter) icons now display correctly via React icon components, and the free plugin registers the "ADAIRE BLOCKS FREE" category so blocks no longer appear uncategorized in the inserter.

## New Features / Improvements

### New Block Icons
- **Hero 1 Block (Free)** – Custom icon added; displays in block picker and admin.
- **Image Composition Block (Free)** – Custom icon added; displays in block picker and admin.
- **Info Grid Block (Free)** – Updated to part5 icon; displays in block picker and admin.
- **Info Grid 2 Block (Free)** – Custom icon added; displays in block picker and admin.
- **Info Grid 3 Block (Plus)** – Custom icon added; displays in block picker and admin.
- **Info Grid 4 Block (Plus)** – Custom icon added; displays in block picker and admin.
- **Card Scroll Block (Free)** – Custom icon added; displays in block picker and admin.
- **Card Scroll Item Block (Free)** – Uses Card Scroll icon; displays in block picker and admin.
- **Footer Block (Free)** – Custom icon added; displays in block picker and admin.

### Block Picker (Inserter) Icons
- Blocks above now use **React icon components** instead of raw SVG from `block.json`, so WordPress does not strip or hide icons in the block inserter.
- Icons are consistent between block management admin and the block picker.

### Free Plugin
- **Block category registration** – The free build now registers the "ADAIRE BLOCKS FREE" category so free blocks appear under that category in the block picker instead of "Uncategorized".
- Implemented in the free-version generator (`scripts/generate-free-version.js`) so future free builds include it automatically.

## Technical Details

### Icon System
- New icons added under `src/icons/new-icons/part5/` and (for free blocks) copied to `src/icons/new-icons/Free Blocks/`.
- `config/block-icon-mapping.json` updated with `blockIconMap` and `newIconMap` entries for hero-1, image-composition, info-grid, info-grid-2, info-grid-3, info-grid-4, card-scroll, footer (container).
- `npm run prebuild` (apply-new-icons.js + update-block-icons.js) generates/updates React icon components and writes SVG into each block’s `block.json`.

### Block Registration
- Blocks now pass an explicit **icon component** (e.g. `Hero1Icon`, `InfoGrid2Icon`) to `registerBlockType()` so the inserter receives a React element and can render the icon.
- `src/icons/index.js` exports `CardScrollIcon` and `ContainerIcon` for use by card-scroll and footer blocks.

## Upgrade Impact

- **No breaking changes.** Existing blocks and content are unchanged.
- **Action required:** None. Icons and category behavior apply automatically after update.
- **Free version:** Regenerate the free build (e.g. `node scripts/generate-free-version.js`) and rebuild to get the new category and icons.

## Backward Compatibility

- All changes are additive. Block JSON and attributes are unchanged except for the `icon` field (SVG string) in `block.json`, which does not affect saved content or front-end output.

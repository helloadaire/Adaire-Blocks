# Adaire Blocks v1.2.1 Release Notes

Version 1.2.1 is a maintenance release that introduces a unified icon mapping system and improves the build process for better maintainability and consistency.

## Developer Improvements

### Unified Icon Mapping System
- **Centralized Configuration**: Created `config/block-icon-mapping.json` as a single source of truth for all block-to-icon mappings
- **Easy Maintenance**: All icon mappings can now be edited in one JSON file instead of scattered across multiple script files
- **Dual Mapping Support**: The configuration file includes both `blockIconMap` (for icon component file names) and `newIconMap` (for SVG source file paths)
- **Build Script Integration**: Both `apply-new-icons.js` and `update-block-icons.js` now read from the unified mapping file

### Build Script Enhancements
- **Centralized Icon Processing**: Updated build scripts to use the unified mapping configuration
- **Improved Error Handling**: Better logging and error messages when mapping file is missing
- **Consistent Mapping Logic**: Both scripts now use the same mapping source, ensuring consistency

### Icon Path Corrections
- **Progress Block Icon**: Fixed icon path from Free tier to Plus tier (`progress-block.svg` → `progress-plus-block.svg`)
- **Tier Accuracy**: Ensured all block icons use correct tier-specific icon files

## Technical Details

### Configuration File Structure
The new `config/block-icon-mapping.json` file contains:
- `blockIconMap`: Maps block directory names to icon component file names (e.g., `"accordion-block": "accordion"`)
- `newIconMap`: Maps block directory names to SVG file paths in `src/icons/new-icons/` directory

### Script Updates
- `scripts/apply-new-icons.js`: Now loads mappings from config file and uses `blockIconMap` to determine correct icon component file names
- `scripts/update-block-icons.js`: Updated to load from config file (already had this functionality, improved error messaging)

### File Changes
- Added: `config/block-icon-mapping.json` - Unified icon mapping configuration
- Modified: `scripts/apply-new-icons.js` - Uses unified mapping
- Modified: `scripts/update-block-icons.js` - Improved error messaging

## Upgrade Impact

- **No Breaking Changes**: All existing blocks continue to work as before
- **No User-Facing Changes**: This is primarily a developer/maintenance improvement
- **Backward Compatible**: Build scripts fall back to default behavior if mapping file is missing
- **No Migration Required**: Existing installations work without any changes

## Developer Benefits

- **Easier Icon Management**: Edit one JSON file instead of multiple script files
- **Better Organization**: Clear separation between block names and icon file names
- **Simplified Onboarding**: New developers can understand icon mappings by reading one configuration file
- **Reduced Errors**: Centralized configuration reduces the chance of mismatched mappings

## Performance Considerations

- No performance impact on frontend or editor
- Build process remains the same speed
- Configuration file loading is minimal overhead

This release improves the developer experience and maintainability of the icon system while maintaining full backward compatibility.

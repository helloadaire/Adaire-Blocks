# Testing Notes - Version 1.2.4

## Pre-Release Testing Checklist

### Block Icons – Block Picker (Inserter)

- [ ] Open block inserter (add block) and locate **Hero 1**, **Image Composition**, **Info Grid**, **Info Grid 2**, **Info Grid 3**, **Info Grid 4**, **Card Scroll**, **Card Scroll Item**, **Footer** blocks
- [ ] Each of the above blocks displays its **custom icon** (not a generic/dashicon) in the inserter
- [ ] Icons are clear and recognizable at default inserter size
- [ ] No broken or missing icon placeholders

### Block Icons – Block Management (Admin)

- [ ] Go to **Adaire Blocks → Block Management** (or equivalent settings page)
- [ ] Verify the same blocks (Hero 1, Image Composition, Info Grid, Info Grid 2, Info Grid 3, Info Grid 4, Card Scroll, Card Scroll Item, Footer) show **custom icons** in the block list
- [ ] Icons match the style of other blocks (e.g. Accordion, Button, Testimonial)

### Free Plugin – Block Category

- [ ] Install/activate the **free** build of the plugin
- [ ] Open the block inserter and expand block categories
- [ ] **ADAIRE BLOCKS FREE** category is present
- [ ] Free blocks (e.g. Hero 1, Image Composition, Info Grid, Info Grid 2, Card Scroll, Footer) appear under **ADAIRE BLOCKS FREE**, not under "Uncategorized"
- [ ] Adding a block from this category works as expected

### Regression – Existing Blocks

- [ ] Existing blocks (Accordion, Button, Testimonial, Tabs, etc.) still show their icons in the inserter
- [ ] No new console errors when opening the inserter or inserting any block
- [ ] Block registration and editor behavior unchanged for blocks not modified in 1.2.4

### Build & Prebuild

- [ ] `npm run prebuild` completes without errors
- [ ] `npm run build` completes without errors
- [ ] Built block.json files for the updated blocks contain an `icon` field (SVG string) for fallback/consistency
